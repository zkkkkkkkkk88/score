# 体彩足球赛事雷达

一个中文足球赛事分析面板，用于展示中国竞彩网公开赛事、实时比分、赛前预测、购买方案、历史方案和命中复盘。

说明：本项目只做信息整理和本地模型分析，不展示赔率，不保证命中，不构成投注建议。赛事、竞彩编号、销售状态和比分来自中国竞彩网公开接口；购买方案和概率为本地模型估算。

## 功能

- 赛事总览：今日重点、每日方案汇总、明日预选池。
- 赛事分析：左侧赛事列表独立滚动，右侧分析固定显示；支持全部、今日、明日、近 7 天和可购买筛选。
- 近 7 天历史赛事：刷新数据时保留最近 7 天已出现过的完赛赛事；如果体彩当前列表不再返回，会从历史购买方案里恢复比赛卡和赛前预测。
- 单场玩法：胜平负、让球胜平负、具体总进球数、比分、半全场；只展示该场真实开放或已归档确认的玩法。
- 赛果对比：玩法卡片上方显示赛前预测，下方显示完赛真实结果，并自动标记正确或错误。
- 购买方案：按二串一、三串一、三串二、四串一、四串二分组；每组最多展示 5 个方案，适配每天大量串单。
- 历史方案：按已完赛和未完赛左右分栏，再按日期和串单类型归类。
- 命中复盘：按日期折叠归档，每一天下面可以点开查看具体购买方案和每场结果。
- 自动刷新：实时服务默认每 10 分钟刷新一次 Sporttery 数据，并重新生成购买方案；接口限流时会继续使用本地缓存并自动退避。

## 本地运行

推荐使用实时服务启动：

```powershell
node scripts/serve-live.js
```

访问：

```text
http://127.0.0.1:4173
```

实时服务会：

- 启动时如果本地缓存已过期，会拉取一次中国竞彩网公开数据。
- 默认每 10 分钟刷新一次，避免过于频繁请求触发接口拦截。
- 提供 `/api/matches` 给前端读取最新数据。
- 数据过期时自动运行 `scripts/update-real-data.js`。
- 如果中国竞彩网返回 403，服务会进入退避等待，并继续展示最近一次成功缓存的数据。
- 当后台接口被 WAF 拦截时，数据脚本会自动启动本机 Chrome，在官方页面环境里采集赛事列表并合并到本地缓存。

## 手动刷新数据

```powershell
node scripts/update-real-data.js
```

如果只是要用现有缓存重新生成近 7 天历史赛事，不访问体彩接口：

```powershell
$env:SCORE_REPAIR_HISTORY_ONLY="1"
node scripts/update-real-data.js
```

可选环境变量：

```powershell
$env:SPORTTERY_PAGE_SIZE="80"
$env:SCORE_UPDATE_INTERVAL_MS="600000"
$env:SCORE_DATA_MAX_AGE_MS="600000"
$env:SCORE_HISTORY_WINDOW_DAYS="7"
$env:SCORE_BROWSER_FALLBACK="1"
$env:PORT="4173"
node scripts/serve-live.js
```

## 静态打开

也可以用普通静态服务查看页面：

```powershell
python -m http.server 4173
```

然后访问：

```text
http://localhost:4173
```

普通静态服务不会自动刷新真实数据。需要实时更新时，请使用：

```powershell
node scripts/serve-live.js
```

## 测试

```powershell
node tests/dashboard-smoke.test.js
```

该测试会检查页面基础渲染、玩法标签、购买方案、历史方案归类、近 7 天历史赛事恢复、命中复盘折叠、比分与总进球一致性等关键行为。

## 数据与复盘规则

- 当前赛事数据写入 `data/matches.json`。
- 每次生成的购买方案会保存到 `planArchive`。
- 完赛后系统会自动判断每个方案和每个单场选择是否正确。
- 未完赛场次保持待复盘状态。
- 近 7 天历史赛事优先使用体彩当前列表；当前列表缺失时，从 `planArchive` 恢复赛前预测和完赛比分。
- 历史命中率只统计项目实际保存过的购买方案，不补造接入前的历史结果。
